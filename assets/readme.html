SAP NetWeaver 7.52 SP04 - TestDrive

This download provides the software required to install and run SAP NetWeaver Application Server ABAP 7.52 SP04 You can explore SAP's current generation ABAP Application server, learn how to develop web applications and get an overview of SAP's client/server technology.

Please bear in mind that a productive SAP System is designed and optimized for mission-critical applications in a business environment with thousands of parallel users.

To find out about the usage rights granted by SAP for the included components as listed below, consult the SAP DEVELOPER CENTER MASTER SOFTWARE DEVELOPER LICENSE AGREEMENT provided.

  • SAP NetWeaver 7.52 SP04
  • SAP GUI for the Java Environment 7.50 and SAP GUI for Windows 7.50 in recent versions
  • SAP Sybase ASE 16.0.2

Requirements

You need to read and accept the Software End User License Agreement

The NW7.52 TestDrive has been successfully tested with recent versions of the following distributions:

  • SUSE SLES
  • Open SUSE
  • Fedora
  • RHEL
  • Ubuntu and Debian: You need to install csh before you start the installation!
  • Other distributions may require manual setup.

It is mandatory that you use a 64-Bit OS version.

Hardware requirements:

  • x86_64 Processor based hardware
  • At least 4 GB RAM plus about 8 GB swap space
  • About 100 GB free disk space for server installation
  • About 2 GB free disk space for client installation

Installation Instructions and First Steps

Step 1: Prerequisites

  • Network configuration of the SAP host operating system: The officially supported setup requires a static IP address for the hostname used for the SAP installation. Using DHCP is possible for this test-drive version, but be aware of potential consequences. If our installation cannot ping the hostname specified when installing, installation and startup of the SAP system will fail. Make sure that the hostname specified during installation is assigned to the local OS and fulfils SAP requirements, most importantly it cannot exceed 13 characters length and also needs to survive reboots. We strongly recommend that you stick to vhcalnplci as hostname used by the SAP system and make sure that this hostname remains pingable after server restart.
  • The firewall must be deactivated.

Step 2: Server Installation and Getting a License

  • Login with your preferred user. You need to have root privileges (sudo or password login).
  • Extract the downloaded rar archive, e.g. by executing unrar commands in the download folder
    $ unrar x TD752SP04.part01.rar
    and optionally
    $ unrar x install.rar
    to overwrite install.sh with an executable version. Please note that we have compressed install.sh separately into install.rar in order to maintain the script's executable permissions.
  • Change into the root directory of the extracted installation
    $ sudo ./install.sh
    • More detailed installation instructions are provided in the following blog post: SAP AS ABAP 7.52 SP04 Step-By-Step Installation Guide
  • Installation will take a while (about 20 minutes - take a coffee).

    • To get a license:

    1. Start the SAP system
      1. Switch to user npladm in the console : $ su - npladm
      2. In SAPLogon, start the SAP system : $ startsap
    2. Log on to the system with client 000 and user SAP* (see table with SAP users below). Open transaction SLICENSE to ascertain your hardware key. Alternatively, you can get your hardware key by executing $ /usr/sap/NPL/SYS/exe/run/saplicense -get.
    3. Request the license key for your test-drive version SAP License Keys for Preview, Evaluation, and Developer Versions. You can also generate a new key for your ASE database if the existing key expires.
      1. Select NPL - SAP NetWeaver 7.x (Sybase ASE) as System ID
      2. Enter your personal data, agree to the License Agreement, and choose Generate.
      3. The website automatically generates a .txt file for this system/key. Download and save this file.
    4. In SLICENSE, install the license file as follows: In the tab Digitally signed licenses, delete the existing license, then choose Install.
    5. The developer access key for user DEVELOPER and installation number DEMOSYSTEM is already in the system and you can start developing in the customer name range (Z*, Y*).
      Please note that all the above steps must be carried out; otherwise, the prepared user key will not work.

Administration of SAP NW ABAP 7.52 Test-Drive

Created directories and a new hostname

  • The installed system creates the following directories:
    • Table 2: Directories
    Directory Size
    /sapmnt ~ 2 GB
    /sybase ~ 50 GB
    /usr/sap ~ 3 GB
  • If you prefer to provide dedicated disk space for the directories shown in table 2, do so before you invoke the script. Please be aware that the used extraction will overwrite dynamic links, so we suggest to use bind mounts in order to persist SAP paths in your host OS /etc/fstab. Assuming that you have made disk space available on directory /sap, /etc/fstab entry /sap/sybase /sybase none bind 0 0 will make the sybase directory (directories must exist!) available at the correct location while occupying disk space on the device you have added for running the TestDrive.
  • The installation does not cater for any hostname requirements, so please make sure that you have understood the requirements before you execute the installation script.

Uninstallation Instructions

If you ever want to uninstall your server you can do this as follows:

  1. Delete the created directories shown in table 2,
  2. Delete the users shown in table 3,
  3. Delete the added lines in the file /etc/services
    (should be the last lines beginning with sap* and sql6 * respectively),
  4. Delete the line containing vhcalnplci in the file /etc/hosts,
  5. If applicable to your Linux version: delete the symbolic link S99_vhcalnplci.sh in the directory /etc/init.d/rc3.d in case of SuSE or /etc/rc3.d in case of Red Hat.
  6. Now restart your network.
